Search Criteria Hints

In general, entering free-form text as search criteria works pretty well.
This is called Global search and includes the use of field-values.
Alternatively, Field search is the use of queries that include field names.

Entering a single word like 'security' or 'kubernetes' (a Global search)
seems to match useful content very well. However, some may prefer more
precise searching. It's helpful to look through the Query Strings
reference linked below for the details.

A few of the most common reminders for queries:
  • A space is an IMPLIED AND operator.
  • Boolean operators (AND, OR, NOT) must be CAPITALIZED.
  • You can use > and < operators with NUMBER and DATE fields.
  • Our fields are listed below.

Field (Data) Schema

  • index: 'videosearch1'
  • fields:
    • title: TEXT
    • subtitle: TEXT
    • description: TEXT
    • event: ATOM (next2016, next2017, next2018, next2019, next2020, solution, accelerate2017)
    • published_date: DATE (full date - no partial matching)
    • published_date> DATE (full date - no partial matching)
    • speakers: TEXT
    • video_id: TEXT
    • duration_min: TEXT
